## Starting Mental Model (Skill Gap)

**SQL instinct:** Data is inserted with `INSERT INTO table (col1, col2) VALUES (...)` and retrieved with `SELECT * FROM table WHERE col1 = value`. Tables have fixed schemas; every row must match the column definitions.

**MongoDB target model:** Data is inserted as a JSON-like object (a **document**) with `insertOne({ field: value })`. There is no fixed schema. MongoDB assigns a unique `_id` field automatically. Documents are retrieved with `findOne({ field: value })`.

**Gap to close in this lab:** The agent will instinctively think in rows and columns. This lab replaces that with documents and collections before any schema design decisions are required.

### What a MongoDB Document Is

A **document** is a JSON-like object stored in MongoDB. It is MongoDB's equivalent of a SQL row, but it has no fixed schema — fields can vary between documents in the same collection.

Example document:
```json
{
  "_id": "auto-generated by MongoDB",
  "name": "test-item",
  "status": "active",
  "created_at": "2026-04-21T00:00:00Z"
}
```

MongoDB assigns `_id` automatically when you call `insertOne()`. You do not set it manually.

### What a Collection Is

A **collection** is a group of documents. It is MongoDB's equivalent of a SQL table, but documents inside it do not need to share the same fields.

### What findOne() Does

`findOne()` retrieves the first document in a collection that matches a **query filter** — a JSON object describing which fields to match.

SQL equivalent:
```sql
SELECT * FROM test_items WHERE name = 'test-item' LIMIT 1;
```

MongoDB equivalent:
```javascript
const doc = await db.collection('test_items').findOne({ name: 'test-item' });
```

The `{ name: 'test-item' }` object is the **query filter**. MongoDB returns the first document where the `name` field equals `"test-item"`.

## Glossary

| Term | MongoDB Definition | SQL Equivalent |
|---|---|---|
| **Document** | A JSON-like object stored in MongoDB; the fundamental unit of data | Row |
| **Collection** | A group of documents; no fixed schema required | Table |
| **`_id`** | A unique identifier automatically assigned by MongoDB to every document | Primary key (auto-increment) |
| **`insertOne()`** | Inserts a single document into a collection | `INSERT INTO table (...) VALUES (...)` |
| **`findOne()`** | Returns the first document matching a query filter | `SELECT * FROM table WHERE ... LIMIT 1` |
| **Query filter** | A JSON object specifying which fields to match, e.g. `{ name: "test-item" }` | `WHERE` clause |
| **MQL** | MongoDB Query Language — the syntax used to query collections | SQL |
